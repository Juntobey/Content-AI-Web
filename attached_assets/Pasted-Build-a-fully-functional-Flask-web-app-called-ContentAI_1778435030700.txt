Build a fully functional Flask web app called ContentAI.

This must not be a static mockup. It must be a working AI content
generator with backend routes, frontend interaction, prompt logic,
and API integration.

Use this folder structure:

contentai/
├── app.py
├── prompts.py
├── requirements.txt
├── templates/
│   └── index.html
└── static/
    └── style.css

Tech stack:
- Python Flask
- HTML, CSS, JavaScript
- Replit AI API (use whatever AI model Replit provides natively)
- Do NOT use OpenRouter, OpenAI, or any external API keys
- Use Replit's built-in AI integration directly
- python-dotenv only if needed

=== BACKEND: app.py ===

- Create Flask app
- Use Replit's native AI client — import however Replit recommends
- Do not require any API keys from the user
- The AI should work out of the box on Replit
- Create GET route "/" that renders index.html
- Create POST route "/generate" that:
    - Accepts JSON with: feature, input, audience, tone, keywords
    - Validates that input is not empty
    - Calls build_prompt() from prompts.py
    - Sends prompt to Replit AI
    - Calculates and returns:
        result (the AI generated text)
        word_count
        seo_score (calculated from keywords used + word count)
        readability (Excellent / Good / Fair)
        tone_match (echo back the tone)
        keywords_used (e.g. "3/5")
        model_used

=== PROMPTS: prompts.py ===

Create build_prompt(feature, user_input, tone, audience, keywords) function.

Support 3 features:
1. blog_post
2. code_explain
3. prompt_library

All prompts must use a CLEAR framework:
- Context, Length, Examples, Audience, Role

Also apply these ethical guidelines to all prompts:
- Use inclusive, respectful language
- Avoid stereotypes and bias
- Do not invent facts or sources
- Keep output practical and structured

Blog Post Prompt requirements:
- SEO-friendly blog post
- TL;DR at the start
- Engaging introduction
- 3 sections with subheadings
- One practical example or analogy
- Short paragraphs (max 2 sentences)
- Conclusion with call to action
- 400-650 words
- Naturally include the provided keywords
- Match the tone and audience

Code Explainer Prompt requirements:
- Identify the programming language
- Quick summary of what the code does
- Step-by-step explanation
- Explain important concepts
- Include a beginner analogy
- Suggest improvements or warnings
- Do not shame the learner

Prompt Library Prompt requirements:
- Generate 5 reusable prompt cards
- Each card must include:
  Name, Department Tag, Task Tag, Tone Tag,
  Context, Length, Examples, Audience, Role,
  Full Prompt Template, Why It Works,
  Quality Score Criteria (1-5)

1. Top navbar:
